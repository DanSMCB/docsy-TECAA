Esta é uma página de exemplo que te mostra como usar este site modelo.

{{% /pageinfo %}}

Para muitos projetos, os utilizadores podem não precisar de muita informação além do que está no [Resumo](/docs/overview/), por isso esta secção é **opcional**. No entanto, se houver áreas em que os teus utilizadores necessitam de uma compreensão mais detalhada de um determinado termo ou funcionalidade para fazerem algo útil com o teu projeto (ou para não cometerem erros ao usá-lo), coloca essa informação nesta secção. Por exemplo, podes querer adicionar algumas páginas conceptuais se tiveres um projeto grande com muitos componentes e uma arquitetura complexa.

Lembra-te de focar-te no que o utilizador precisa de saber, não apenas no que consideras interessante sobre o teu projeto! Se eles não precisam de compreender as tuas decisões de design originais para usar ou contribuir para o projeto, não as incluas, ou coloca os teus documentos de design no teu repositório e faz link para eles. De forma semelhante, a maioria dos utilizadores provavelmente precisará de saber mais sobre como as funcionalidades funcionam em uso, em vez de como são implementadas. Considera uma página de arquitetura separada para informações mais detalhadas sobre implementação e design de sistemas, que os potenciais contribuidores do projeto possam consultar.
