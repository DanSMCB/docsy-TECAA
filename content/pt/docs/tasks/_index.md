Pensa nas funcionalidades e casos de uso do teu projeto. Usa-os para escolher as
tuas tarefas principais. Cada caso de uso granular (ativar x, configurar y) deve
ter uma página de tarefas correspondente ou uma secção de página de tarefas. Os
utilizadores devem conseguir consultar rapidamente as tuas tarefas principais
quando precisarem de descobrir como fazer uma coisa específica, em vez de ter
que procurar as instruções num tutorial maior ou exemplo. Pensa nas páginas de
tarefas como um livro de receitas com diferentes procedimentos que os teus
utilizadores podem combinar para criar algo mais substancial.

Podes dar a cada tarefa a sua própria página, ou agrupar tarefas relacionadas
numa página, como tarefas relacionadas com uma funcionalidade específica. Além
de agrupar tarefas relacionadas em páginas únicas, também podes organizar páginas
de tarefas em pastas aninhadas com uma página índice como visão geral, como se
vê neste site de exemplo. Ou, se tiveres um pequeno conjunto de documentação
como o [Guia do Utilizador Docsy](https://docsy.dev/docs/) sem páginas de Tutoriais
ou Conceitos, considera adicionar as páginas específicas das funcionalidades
ao nível superior da documentação, em vez de numa secção de Tarefas.

Cada tarefa deve fornecer ao utilizador:

- Os pré-requisitos para esta tarefa, se existirem (isto pode ser especificado no
  topo de uma página com múltiplas tarefas, se forem iguais para todas as tarefas
  da página. "Todas estas tarefas assumem que compreendes.... e que já tens....").
- O que esta tarefa realiza.
- Instruções para a tarefa. Se envolver editar um ficheiro, executar um comando
  ou escrever código, fornece exemplos em formato de código para mostrar ao
  utilizador o que fazer! Se houver múltiplos passos, apresenta-os como uma lista
  numerada.
- Se apropriado, ligações para páginas relacionadas de conceitos, tutoriais ou
  exemplos.
